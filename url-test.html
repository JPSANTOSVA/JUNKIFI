Diagnosing why junkifi.com/qrtruck returns a 404 error

Problem Detected

You're getting a 404 error when accessing junkifi.com/qrtruck even though you've created qrtruck.html.

Why This Happens

Most web servers don't automatically remove the .html extension from URLs. When you visit junkifi.com/qrtruck, the server looks for a file with that exact name, not qrtruck.html.

Possible Solutions:

  • Option 1: Access the file with the full extension: junkifi.com/qrtruck.html
  • Option 2: Configure your server to rewrite URLs (requires server access)
  • Option 3: Create a directory named qrtruck with an index.html file inside

Server Configuration Examples

If you have access to your server configuration, you can set up URL rewriting:

For Apache servers (.htaccess):

RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-d
RewriteCond %{REQUEST_FILENAME}\.html -f
RewriteRule ^(.*)$ $1.html [L]

For Nginx servers:

location / {
    try_files $uri $uri.html $uri/ =404;
}

Quick Fix

For now, use the full HTML extension in your QR codes and marketing links:

  • Use junkifi.com/qrtruck.html instead of junkifi.com/qrtruck
  • Use junkifi.com/QRShop.html instead of junkifi.com/QRShop
  • Use junkifi.com/QRFB.html instead of junkifi.com/QRFB
  • Use junkifi.com/QRGad.html instead of junkifi.com/QRGad

This will work immediately without any server configuration changes.
